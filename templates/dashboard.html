              ⚙
              Como ligar o webhook na Evolution
            
            

              

                No painel da Evolution costuma dar só para colar a URL e marcar
                quais eventos enviar — não precisas de mais opções lá. O resto (secret,
                findGroupInfos) fica do lado deste servidor / Pulseboard.
              

              
    
                
  1. 
                  URL (HTTPS), caminho fixo:
                  …/evolution-webhook no mesmo host que o teu contentor (Easypanel), na porta
                  exposta do webhook Meta (ex. 8080) ou da
                  Pulseboard (ex. 8091), conforme o que tiveres mapeado.
                
    2. 
                
  2. 
                  Secret: define EVOLUTION_CATALOG_WEBHOOK_SECRET no Easypanel
                  (Ambiente). Na Evolution, cola na URL o segredo na query (sem
                  precisar de cabeçalhos), por exemplo:
                  …/evolution-webhook?catalog_secret=TUASENHAQUI
                  (usa urllib.parse.quote se a senha tiver &, #, espaços,
                  etc.). Alternativas aceites pelo servidor: parâmetro secret=, ou cabeçalhos
                  X-Webhook-Secret / Authorization: Bearer se a tua build os suportar.
                  Em dev sem secret: EVOLUTION_CATALOG_ALLOW_INSECURE=1 (ver ENV_TEMPLATE.txt).
                
    3. 
                
  3. 
                  Eventos úteis para aparecerem grupos @g.us: mensagens em
                  grupo (ex. MESSAGES_UPSERT / messages.upsert) e, se existirem na
                  lista, eventos de chats/grupos (ex. CHATS_UPSERT, CHATS_UPDATE).
                  Eventos só de aplicação ou chamadas não catalogam grupos, mas não atrapalham.
                
    4. 
                
  4. 
                  Activar o grupo no catálogo: é obrigatório enviar no grupo a
                  frase Ativar grupo (maiúsculas e acentos são aceites na detecção). Só depois disso
                  o monitoramento desse JID fica ligado e as mensagens passam a actualizar a ficha do grupo.
                  Mensagens suas também contam se a Evolution incluir messages.upsert com
                  fromMe no webhook.
                
    5. 
                
  5. 
                  Carga no servidor: usa Ligar escuta / Pausar escuta
                  abaixo. Em pausa, o teu URL continua a responder 200 à Evolution mas não grava nem
                  chama a API de nomes — útil quando há muito tráfego.
                
    6. 
              

            

          


          

            

              

                Escuta ativa

    Filtros de perguntas (Meta Lead)
    
          
        
    
        
    
          
    
            Estas regras removem linhas do bloco de respostas do lead (o que vai em
            {{respostas}} / {{respostas_filtradas}} nos templates). Elas não
            filtram pelo texto que o utilizador escreveu na resposta — apenas pelo nome interno do campo
            (a “chave” que vem do Meta, por ex. email, phone_number, rótulo em português,
            etc.).
          
    
          
      
            
    • 
              Nome exato — remove a pergunta se o nome normalizado (minúsculas, sem espaços extras)
              for igual à tag. Útil quando sabes a chave exacta.
            
      • 
            
    • 
              Contém — remove se o nome da pergunta contiver o trecho indicado (também em
              minúsculas). Útil para agrupar variações (utm_, referencia, etc.).
            
      • 
            
    • 
              Regex — aplica re.search ao nome da pergunta (case-insensitive). Se a
              expressão for inválida, o backend ignora essa regra silenciosamente — valide antes de contar com ela.
            
      • 
          
    
          
    
            Globais vs cliente: na aba Templates, “Filtros globais” somam-se às regras deste cliente.
            Se a mensagem ainda mostrar dados “cru”, confirme se o template usa {{respostas_raw}} em vez
            de {{respostas}}.
